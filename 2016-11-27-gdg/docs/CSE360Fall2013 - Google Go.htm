      What is Go?
      Go (also known as Golang) is a compiled, open source, concurrent, garbage-collected statically typed language developed by Google. On September 21 of 2007, Robert Griesemer, Rob Pike and Ken Thompson came together and started sketching ideas and goals for this new language. The language became a public open source project on November 10, 2009 and was released on March 28, 2012 [1] .
      The syntax of the language reveals itself to be similar to C and encourages a conventional imperative programming style [2] . Go aims to provide a stable foundation for creating reliable products by including a language specification, standard libraries, and custom tools. It is also a multi-purpose language appropriate for a broad spectrum of uses including web programming, mobile programming and systems programming [3] .

      package main
 
import "fmt"
 
func main() {
    fmt.Println("Hello, World")
}


      Origins of Go

      On September 21, 2007, Griesemer, Pike and Thompson of Java HotSpot virtual machine and V8 Javascript engine decided to start sketching ideas for a new language out of frustration with the pace of building software [3] . Pikes frustration over the slow compilation process for Googles large software systems led him to want to start from scratch to write the kind of programs we need to write here at Google in a way that the tools could be really efficient and the build cycles could be very short[3] . Within a year, Ian Taylor, Russ Cox, Adam Langley and Jini Kim joined Pike, Griesemer and Thompson in their startup of Go[3] . It started out as a 20 percent project time that Googles engineers are given to use however they choose for undirected experimentation but by mid-year of 2008, the language had become a full-time project and had settled enough to attempt a production compiler. Go became a public open source project on November 10, 2009 and eventually, after a couple years of active design and development, Go was considered stable enough to be formally released on March 28, 2012[1] . Since the release date, people from the community have contributed a tremendous amount of ideas, discussions, and code to the project.

      (From left to right) Robert Griesemer, Rob Pike and Ken Thompson

      Major Goals of Go

      The initial reason for the startup of the programming language Go was because Pikes frustration from the slow pace of building software. He says,

      In Google we have very large software systems and we spent so long literally waiting for compilations, even though we have distributed compilation and parallelism in all of these tools to help, it can take a very long time to build a program. Even incremental builds can be slow. And we looked at this and realized many of the reasons for that are just fundamental in working in languages like C and C++, and we needed a different approach. We also decided the tools that everybody used were also slow. So we wanted to start from scratch to write the kind of programs we need to write here at Google in a way that the tools could be really efficient and the build cycles could be very short[3] .

      The goals of the Go project were to eliminate the slowness and clumsiness of software development at Google to make the process more productive and scalable. It was designed by and for people who write, read, debug, or maintain large software systems. It was designed and developed to make working in this environment more productive by including built-in concurrency, clear dependencies, clear syntax, clear semantics, composition over inheritance, easy tooling, garbage collection, rigorous dependency management, the adaptability of software to grow, and robustness across the boundaries between components [4] . Go attempts to reduce the amount of typing (in both senses of the word). Throughout its design it aims to reduce clutter and complexity by having no forward declarations or header files, by declaring everything at once, and by making initialization expressive, automatic and easy to use[4] .

      Features of Go

      Go is Built with todays computing environments in mind, including:
      • Speed
      • Simplicity
      • Multiprocessing
      • Networking
      • Web applications
      The semantics of Go statements is generally C-like so that it is easy for newcomers to start using Go quickly.
      • Go resembles a mix of C and Python.
      • Go is a statically-typed, procedural language.
      Compiling in Go is very fast, even in large programs because strict dependency and package import rules are enforced.
      • Go does not use dependency trees like C/C++ that may re-import the same files many times, and so compilation in Go may be many times faster than a similar C/C++ program
      • Unused package imports are considered compile-time errors.
      • When the compiler executes an import clause, it only imports the already-compiled object file.
      • Remote packages can be imported over a network, for example, from git repositories.[5]
      Go is garbage collected but gives the programmer tools to control garbage collection overhead.

      Go has fundamental built-in concurrency
      • A criticism of C++ and Java are that they lack fundamental concurrency.
      • Goroutines are like threads and are executed concurrently, but are not necessarily run in separate threads. The programmer does not have to think about it as it is handled by Go.
      • Gos concurrency model is influenced by CSP (Communicating Sequential Processes), in that component processes operate independently and interact with each other through sequential or parallel message passing.[6]
      Go functions can return multiple values, which is useful for returning a function result and error code.[7] 
      finfo, err := os.Stat("filename.txt")
if err != nil {
    // no such file or dir
    return
}
...
      Public & Private visibility are built into the object names of everything - variables, types, functions, methods, constants and fields, which makes identifying object visibility very easy for programmers.
      • Public visibility starts with an uppercase letter.
      • Private visibility starts with a lowercase letter.
      Instead of inheritance and subclasses, Go uses interfaces.
      • Inheritance must be considered and designed early, and can be difficult to change once a significant amount of code is written.
      • Polymorphism is handled through interfaces.
      • There are no classes.
      • There are no namespaces.
      Variables are always local to the package.
      • Go has built-in Array, Map and String types.
      • All parameters are passed by value or by pointer.
      • There is no function overloading. Functions are identified by name only, not by signature (type) of the function. A type can never have two functions with the same name.
      • There are pointers, but no pointer arithmetic.
      • Array bounds are always checked.
      There are tools that are not considered part of the Go language but have been influential on it.
      • gofmt is a tool that is constantly run to enforce proper formatting and enforces a single coding style on everyone. [8]
      • gofix is a tool for refactoring code and code trees to update and/or deprecate code. It has been used on the Go source to globally rewrite code and depricate functions. [9]

      Criticisms of Go

      The name Go is difficult to search for. Golang is the preferred way to google.

      Go is still very young and libraries are still very limited.

      As of September, 2013, Go is ranked as the 49th most popular language in the Tiobe Programming Community popularity index [10]

      Go is not purely memory safe in concurrency
      • Learn to think of message passing as a form of ownership. "Don't communicate by sharing memory, share memory by communicating."[6]
      Go has limited exception handling
      • Return error codes from functions instead, which is easy thanks to multiple return values.
      • There is a panic/recovery system for truly exceptional failures.
      There is no implicit number conversion.
      There are no assertions.
      There are no generics.
      Go does not support default function arguments.

      There are many forms of variable declaration syntax[4]
      x int, y int
 
// is equivalent to:
x, y int
 
// and is equivalent to:
var x int
var y int
 
// and all can be initialized in different ways...
      Since Go is still such a new language, the implementation still needs work and there's still many feature requests to be added.[11]


      Who is using Go?


      Go programming language originally caught the attention of C and C++ hackers, who decided to play along and see the advantages that the language would offer them. On the other hand, Google is using Go internally in several programs deployed in production inside Google. The server behind http://golang.org is an example of that, the godoc document server is running in a production configuration on Google App Engine. Go also is being used in the Vitess system for large scale SQL installations and Googles download server, dl.google.com, delivering Chrome binaries and apt-get packages, which is other large installable [12] .



      Apps using Go

      Go is appropriate for a broad spectrum of uses, including Web programming, mobile programming and systems programming. Rob Pike, one of the three founders of the Go project said that the language was based in ideas of what they thought systems programming should be like. Since the language presents port to ARM processors, it allows Go code to run in Android phones[3] . Therefore, applications for Android phones can be written using Go. Go Programming language also allows people to write their Google Cloud Messaging application servers[13] . Moreover, there are many programmer lovers developing some good tools and apps using go. Some of them are listed below:



      • Hatcog is a text-based IRC client, which plays well with tmux, or any other window manager. It was created to allow management of chat from different tmux windows, using the same IRC connection. The Hatcog client is in Python3/curses, and the server in Go.
      • Doozer is a highly available consistent store for small amounts of very important data, which is written using Go. If the data changes, it is possible to notify connected clients immediately (no polling), which makes it ideal for data that is frequently updated and in the case of clients that want real-time updates. Doozer is good for name service, database master elections, and configuration data shared between several machines .
      • StatHat is a tool that tracks statistic and events in your code. Entering some number to StatHat it generates fully embeddable based on the data entered. StatHat also keeps you alert when specific triggers occur sending daily email reports .
      • SmartTwitter is a Facebook application that reposts an individual Twitter status update to the same person Facebook profile. The application allows people to filter retweets, mentions, hashtags, replies and more[14] .





      Examples

      Functions in Go start off really simply, but they provide a few simple extensions. They let you do all sorts of things. Bellow is a factorial function in Go [15] .


      func Factorial(x int) int {
    if x == 0 {return 1;}
    else {return x * Factorial(x - 1);}
}

      Go extends that by adding support for named return values. You can declare the return value as a variable in the function header, and then assign values to that variable. When the function returns, the last value assigned to the return variable is the return value [15] . Therefore factorial could be written as:

      func Factorial(x int) (result int) {
    if x == 0 {result = 1;}
    else {result = x * Factorial(x - 1);}
    return;
}
 

      Go is not object-oriented in a traditional way; however it implements that can be seen as a limited form of object-orientation using another extension to functions. Either type aliases, or structure types can define new types. Methods can be declared for any type at all in the module where it is defined. A method is just a function that has a parameter proceeding the function name[15] .

      For example, a basic linked list could be implemented as:      [15] 
      type IntList struct {
    IntList* next;
    val int;
}
 
func (IntList* s) SetNext(IntList* n)
{s.next = n;}
 
func (IntList* s) GetNext() IntList*
{return s.next;}
 
func (IntList* s) SetValue(v int)
{s.val = v;}
 
func (IntList* s) GetValue() int
{return s.val;}


      Go is open source

      By definition open source is pertaining to or denoting software whose source code is available free of charge to the public to use, copy, modify, sublicense, or distribute[16]

      A large part of development and bug discovering is done by developers outside of Google.

      If you have discover any bugs in the code or in the language documentation you can file a ticket to the message board being. Go encourages any and all issues to be reported no matter how small.[17] You can access this site here: https://code.google.com/p/go/issues/entry

      Community contributions to the libraries are also greatly encouraged. The first stage in the contribution process is to discuss your design. This gives the community to make sure your design lies inside of the languages goals. It also helps to prevent duplication of ideas [18] .

      The next step is testing redux. Before the code is submitted for review you must test and make sure that your contributions will not break any of the previously developed packages.[18]



      The code is all managed by Mercurial in a repository. When code is ready to be contributed it is pushed to the repo.

      All code additions must be reviewed, no matter who made the contribution. The reviewer may make suggestions, reject, or ok the code. Once this process is done only certain members within the community can officially commit the changes to the official repository.
      For the new V. 1.1 google accepted 2600 changes made by 161 different contributors[19]

      The Go Nuts mailing list, which can be found here: https://groups.google.com/forum/#!forum/golang-nuts

      This is essentially a general discussion forum which can include anything from best practices to questions on implementation

      A main wiki is also offered here:

      https://code.google.com/p/go-wiki/wiki/Projects

      The wikis goal is to list everything possible about the language.

      There is also a Go blog which can be found here:

      http://blog.golang.org/

      Version 1.1
      Google reports version 1.1 should run 30 to 40 percent faster than the previous versions[20]

      This version included mostly optimizations in the compiler and linker, garbage collector, go routine scheduler, map implementation, and parts of the standard library[20]

      Some of the changes include
      • Compilers perform more in-lining[20]
      • Garbage collector has been made more parallel, reducing latency[20]
      • A more precise garbage collector which can free up needed heap space[20]
      Changes would not be possible if not for the open source community.
